## 📊 프로젝트 스펙

### 📌 개발 언어

- **Front-End** : JavaScript
- **Back-End** : Java 21

### ⚙️ 상세 스펙

- **Framework** : Spring Boot
    - **ORM** : JPA(Java Persistence API)
    - **View-Template** : Thymeleaf
- **Datebase**
    - **RDBMS** : MySQL
    - **NoSQL** : Firebase
- **AI**
    - **LLM(Large Language Model)** : LangChain4j

            1. 데일리 루틴 데이터를 배열로 받는다.
            2. 데이터를 받아서 DB에 업데이트 후 저장된 후 데일리 루틴 배열을 반환

        1. 화면 로드 후 서버에 데이터 요청 (Front → Back → DB → Back → Front)
        2. 요청한 REST API return 값을 바인딩
    4. 데일리 루틴 쉬어가기 기능
        1. 쉬어가기 버튼 클릭시 데일리 루틴 상태값 업데이트 ex) `enum.PASS`
        2. ID만 넘기기

        3. 완료 시 타이머 시간이 다 채워져야 한다.
    6. 데일리 루틴 완료 취소 기능
        1. “취소” 버튼 클릭 시 하단에 있던 데일리 루틴이 기존 데일리 루틴으로 옮겨지기
        2. DB에 업데이트, ID만 넘기기
5. 순공 시간 타이머
    1. 데일리 루틴을 수행하기 위한 시간을 체크하는 타이머
    2. 카운트 다운 방식으로 시간 계산(포모도로 타이머에서 사용한 시간 만큼 빼기)
    3. 목표 시간내에 완료하지 못하면 알람을 주기
    4. 포모도로 타이머를 활용해 데일리 루틴 수행
        1. setTimeout 쓰는방식: [https://baeharam.netlify.app/posts/javascript/JS-setTimeout을-멈춰보자](https://baeharam.netlify.app/posts/javascript/JS-setTimeout%EC%9D%84-%EB%A9%88%EC%B6%B0%EB%B3%B4%EC%9E%90)
        2. Front에서 시간 계산 및 타이머 이벤트를 구현
            1. 데일리 루틴에 얼마나 시간을 썻는지 DB에 업데이트
            2. 프론트에서 시간 정보는 일단 localstoreage에 저장
            3. 일시정지 및 완료가 되면 DB로 전송
            4. 일시정지가 되면 로컬스토리지에 저장된 타임과 업데이트할 타임을 비교했을 때 특정 시간 ex)1분 이상 차이가 난다면 DB에 업데이트
            5. focus out 이벤트 리스너(시간이 여유가 생기면)
                1. 페이지에서 포커싱이 빠지면 다른일을 한다는 것으로 감안해서 타이머 정지/시작 이벤트
        3. 포모도로 타임 셋 설정
            1. 25분/5분 쉬기, 40분/10분 쉬기 … 유명한 예시 템플릿으로 일괄 설정
        4. 시작 이벤트
            1. 시작 버튼을 프론트에서 클릭 시 카운트 다운 진행
        5. 정지 이벤트
            1. 정지 버튼을 프론트에서 클릭 시 카운트 정지 및 (필요시)DB에 업데이트
        6. 종료 이벤트
            1. 종료 버튼을 클릭 시 타이머를 멈추고 DB에 업데이트 후 포모도로 타이머 종료
            2. (타이머용 화면이 있을 경우에 종료 버튼 클릭 시 이전 화면으로 이동)

2. 개발자 기술 면접 질문
    1. 해당 카테고리의 문제가 약 5개 정도 나온다.
    2. 사용자가 카테고리 선택
    3. 사용자는 input에 답변을 쓴다.
    4. 다음 문제로 넘어가서 다시 답변을 쓴다.
    5. 전체 완료 후 답을 보는 방식
        1. `LLM 플래너 가이드로 쓰기에 어렵다면 문제에 대한 모든 답변을 LLM에게 던져서 피드백 받는 방법도 있음`
    6. 면접 질문은 관리가가 관리자 페이지에서 관리
    7. 카테고리를 선택하면 서버로 요청을 한다.
        1. 해당 카테고리에 대한 문제들을 랜덤하게 뽑아서/난이도 별로 개수를 정한 후 뽑기 return
3. 마이 페이지
    1. 순공 시간 그래프
        1. 총 순공 시간(모든 데일리 루틴에 내가 얼마나 시간을 썻는지에 대한 총 합)
        2. 카테고리 별 시간을 볼 수 있어야 한다.
        3. DB에서 해당 사용자의 전체 데일리 루틴 데이터를 전달해주면 Chart.js에서 카테고리별 구분 가능
            1. 사용자의 전체 데일리 루틴 데이터가 많을 경우 → 총 시간의 합 조회에 문제가 생길 수 있다. → DB에서 총 합 시간을 계산 후 return 하는 방식으로 개선 가능 혹은 총 합을 따로 사용자의 테이블에 속성을 추가해서 관리 가능
    2. 데일리 루틴 완료 비율
        1. 전체 총 완료 비율
        2. 카테고리별 완료 비율
        3. 일일 완료 비율
            1. 시간이 허락한다면 주간 별 완료 비율
    3. 순공 시간, 데일리 루틴 완료 비율 데이터 저장 기간

### 🧑‍💼 관리자 기능 및 역할

    1. 면접 질문, 문제 관리
